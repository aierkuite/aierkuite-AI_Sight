# 落地页开场：滚轮整屏翻页（一次手势一页）

> 权威方案：仓库根目录 `plan-paged-scroll.md`（已与用户逐条敲定）。本 PRD 仅抽取需求与验收标准，技术细节见 `design.md` 与方案 §3–§7。

## Goal

把电影开场 `CinematicLanding` 从「连续平滑滚动长卷（Lenis + 380vh）」改成「离散整屏翻页」：共 5 页（`0=Hero, 1=看得见, 2=听得懂, 3=边想边答, 4=CTA`），落点永远在某一页；一次连续滚轮手势最多翻一页，停顿后再滚才翻第二页；滚轮/键盘/触摸统一一次手势一页；末页用两段式「落地页掭起」进入工作台。弃用 Lenis 与 380vh。

## Requirements（来自方案 §2，已敲定）

* 离散整屏翻页，5 页，锚点 `ANCHORS = [0, 0.32, 0.52, 0.7, 1.0]`（对照 CSS 已核验，§4.3）。
* 完全接管导航：去真实滚动与滚动条，JS rAF 在锚点间补间 `scrollProgress`。
* 手势判定 = 动画锁（~0.8s 翻页期间忽略滚轮）+ 静默再武装（动画结束后需滚轮静默 ~120ms 才允许下一次翻页）→ 一次连续滚动只翻一页。
* 输入统一：滚轮 + 键盘（`↓/→/PageDown/空格`=下页，`↑/←/PageUp`=上页，`Home/End`=首/末页）+ 触摸（上滑=下页，下滑=上页）。
* 过渡 ~0.8s `easeInOutCubic`；reduced-motion 用**完全相同**的滑动（不分支，旧的 reduced-motion 原生滚动分支整段删除）——靠 JS rAF 绕开全局 `prefers-reduced-motion` 把 transition 压成 0.001ms 的规则。
* 末页两段式掭起：page4 下滑①→peek 预览（**工作台从屏幕底部升起、初始露出 ~20%、偏暗**，落地页 CTA 仍在）；下滑②→工作台升满并由暗转亮、落地页整体淡出 → `setStage('workspace')`；预览态上滑→退回 CTA 页、工作台滑回隐去。
* 两端钳制：首页再上滑无动作；末页非 peek 边界由两段式接管。
* 进入/重播一致性：每次挂载（含 `replayIntro`）重置 page0 / `scrollProgress=0` / `peek=0`。

## Acceptance Criteria（= 方案 §9 人工复核清单）

* [ ] 滚轮：一次连续滚（含按住不放/快速连滚）只翻一页；停顿后再滚才翻第二页。
* [ ] 触控板：一次惯性滑只翻一页，不连翻。
* [ ] 5 页落点干净：每页只有对应一幕清晰可见，3D 粒子随翻页平滑演化、无跳变。
* [ ] 键盘 `↓/↑/PageDown/PageUp/空格/Home/End` 行为正确，一次一页；按住不连翻。
* [ ] 触摸上/下滑各翻一页（移动端或 DevTools 触摸模拟）。
* [ ] 无滚动条、页面不可自由滚动。
* [x] 末页下滑①工作台从底部升起、初始露出 ~20%、偏暗；下滑②升满转亮并进入；预览态上滑退回。**揭示方向/幅度/暗度已由用户现场确认**。
* [ ] 进入工作台无闪跳；摄像头/语音功能正常（hook 绑定未受影响）。
* [ ] reduced-motion 下翻页仍是滑动、非硬切；场景走 2D 兜底。

## Validation

在 `frontend/` 下：`npm run lint` → `npm run build`（`tsc --noEmit && vite build`，零类型错误）→ `npm run test`（既有测试全绿，不涉及落地页）→ `npm run dev`（人工复核）。

## Review Gate

实现 + 自动校验通过后**先不提交**：起 `npm run dev`，按上面验收清单逐项自查，并暂停让用户现场看运行效果（尤其末页掭起的揭示方向/幅度），用户确认后才提交。

## Out of Scope（方案 §11）

* 不改工作台 `shell` 内部布局与任何 chat/设备逻辑。
* 不引入路由/URL hash、不做翻页页码指示器。
* 不改 3D 场景形态编排（仅复用其对 `scrollProgress` 的现有读取）。
* 不为 reduced-motion 单独设计过渡（与常规一致）。
